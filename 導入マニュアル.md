# チームメンバー募集アプリ 導入マニュアル

このアプリをあなたのチームで使えるようにするための手順書です。
プログラミングの知識がなくても、この手順通りに進めれば設置できます。

---

## 完成イメージ

- チームの紹介・費用・練習日程を外部に公開できる
- 体験入団の申し込みをWeb上で受け付けられる
- 管理画面から申込者の一覧・ステータスを管理できる
- 申込があると、指定のメールアドレスに通知が届く

---

## STEP 1：チーム情報を設定する

`lib/team-config.ts` をテキストエディタで開いて、各項目をあなたのチームの情報に書き換えてください。

```ts
const teamConfig = {
  teamName: "○○少年野球チーム",        // チームの正式名称
  teamShortName: "○○メッツ",           // 短いチーム名（メールの件名などに使用）
  sport: "野球",                        // スポーツ種別（野球・サッカー・バスケなど）
  catchphrase: "一緒に野球しよう！",    // トップページの大見出し
  subCatchphrase: "小学生大募集！初心者歓迎！", // トップページの小見出し
  target: "小学生",                     // 対象（小学生・中学生など）
  trialNote: "体験は1ヶ月を目処に...", // 申込フォームの注意書き
  siteUrl: "https://あなたのURL.vercel.app", // 後でVercelのURLに変更する
  localStorageKey: "チーム名_chat_id",  // 英数字で一意の文字列（例: abc_chat_id）
  copyrightYear: "2026",               // 年号
};
```

## STEP 2：画像を差し替える

`public/` フォルダの中にある以下の2ファイルを、あなたのチームの画像に差し替えてください。
**ファイル名はそのまま（変えないでください）。**

| ファイル名 | 内容 | 推奨サイズ |
|-----------|------|-----------|
| `public/logo.png` | チームのロゴ画像 | 横300px × 縦95px 程度 |
| `public/hero.png.png` | トップページの背景画像 | 横1200px以上 |

## STEP 3：Supabaseのデータベースを作る

### 3-1. アカウント作成
1. [supabase.com](https://supabase.com) でアカウントを作成する
2. 「New project」でプロジェクトを作成する（プロジェクト名は何でもOK）

### 3-2. テーブルを作成する
Supabaseの管理画面 → 「SQL Editor」を開いて、以下のSQLをコピー&ペーストして「Run」を押す。

```sql
-- 体験申込者テーブル
create table applicants (
  id uuid primary key,
  child_name text not null,
  child_grade text not null,
  parent_name text not null,
  trial_date text,
  message text,
  status text default '未確認',
  created_at timestamp with time zone default now()
);

-- チャットメッセージテーブル
create table messages (
  id uuid primary key default gen_random_uuid(),
  applicant_id uuid references applicants(id),
  sender text not null,
  content text not null,
  created_at timestamp with time zone default now()
);

-- 体験会日程テーブル
create table trials (
  id uuid primary key default gen_random_uuid(),
  date date not null,
  start_time time not null,
  location text,
  note text,
  created_at timestamp with time zone default now()
);

-- チーム情報テーブル
create table team_info (
  id integer primary key default 1,
  description text,
  founded_year text,
  member_count text,
  practice_days text,
  practice_location text
);

-- 設定テーブル（通知メールアドレスなど）
create table settings (
  key text primary key,
  value text
);

-- 通知メールアドレスの初期値を登録（あなたのメールアドレスに変更してください）
insert into settings (key, value) values ('notification_email', 'your@email.com');

-- チーム情報の初期データを登録
insert into team_info (id, description, founded_year, member_count, practice_days, practice_location)
values (1, 'チームの説明を入力してください', '2000年', '20名', '土・日曜日', '○○公園');
```

### 3-3. APIキーを確認する
Supabaseの管理画面 → 「Project Settings」→「API」を開いて、以下の2つをメモする。

- **Project URL**（`https://xxxx.supabase.co` という形式）
- **anon public key**（長い文字列）

## STEP 5：GitHubにコードをアップロードする

1. [github.com](https://github.com) でアカウントを作成する
2. 「New repository」でリポジトリを作成する（名前は何でもOK、Privateを推奨）
3. 以下のコマンドを実行してコードをアップロードする

```bash
git init
git add .
git commit -m "初回コミット"
git branch -M main
git remote add origin https://github.com/あなたのユーザー名/リポジトリ名.git
git push -u origin main
```

> **注意：** `.env.local` ファイルは絶対にアップロードしないでください（`.gitignore` に含まれているので通常は問題ありません）。

---

## STEP 6：Vercelで公開する

1. [vercel.com](https://vercel.com) でアカウントを作成する（GitHubアカウントで登録するとスムーズ）
2. 「New Project」→ GitHubのリポジトリを選択する
3. **「Environment Variables」に以下の4つを設定する**（これをしないと動きません）

| 変数名 | 値 |
|-------|-----|
| `NEXT_PUBLIC_SUPABASE_URL` | STEP 3-3 でメモした Project URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | STEP 3-3 でメモした anon public key |
| `RESEND_API_KEY` | STEP 4-1 でメモした APIキー |
| `ADMIN_PASSWORD` | 管理画面のパスワード（好きな文字列） |

4. 「Deploy」を押す
5. デプロイ完了後、発行されたURLを確認する（例：`https://○○.vercel.app`）

### 6-1. URLを team-config.ts に反映する
`lib/team-config.ts` の `siteUrl` を、発行されたVercelのURLに更新する。

```ts
siteUrl: "https://○○.vercel.app",  // ← 発行されたURLに変更
```

変更後、GitHubにpushするとVercelに自動反映される。

```bash
git add .
git commit -m "サイトURLを更新"
git push
```

## よくある質問

**Q. 管理画面のURLは？**
`https://あなたのURL.vercel.app/admin` です。パスワードはVercelの環境変数 `ADMIN_PASSWORD` に設定した値です。

**Q. 通知メールが届かない**
Supabaseの `settings` テーブルの `notification_email` の値を確認してください。

**Q. 体験会日程を登録したい**
管理画面 → 「体験会管理」から登録できます。

**Q. チーム紹介の文章を変えたい**
管理画面 → 「コンテンツ管理」から編集できます。
